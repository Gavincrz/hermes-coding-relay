# Agent 指令

保持本文件精简。设计、任务和决策分别放在 `DESIGN.md`、`docs/TASKS.md`、`docs/DECISIONS.md`。

## 开始前

编码前先读：

- `DESIGN.md`
- `docs/TASKS.md`
- `docs/DECISIONS.md`

工作规则：

- 每次只处理 `docs/TASKS.md` 中的一个任务。
- 如果用户提出的新需求不在 `docs/TASKS.md`，先补任务定义，除非用户明确说不要。
- 如果当前需求还在讨论、澄清或评审阶段，先分析和收敛方案，不直接改代码。
- 如果任务跨度过大，先拆成更小的可交付任务，再开始实现其中一个。

## 规则

### 架构边界

- Hermes 源码零修改；所有能力通过本插件实现。
- 第一版只支持 Codex；不要为了未来多 agent 提前做重抽象。
- 事件解析、会话状态管理、平台输出格式化必须分层；禁止写成一个大函数。
- `codex_thread_id` 是恢复会话的唯一外部真相；不要再发明语义重复的会话 ID 并混用。
- 第一版优先做“可靠 relay”，不把 scope 扩展成通用 agent 平台。

### 状态与运行时

- 运行态数据只能写入 `run/`。
- `run/` 必须加入 `.gitignore`。
- `run/` 只存运行态数据，不存源码配置真相。
- 删除 `run/` 后，项目应仍可通过已有源码和配置重新启动。
- 不要在仓库其他位置写入临时状态文件、日志文件或调试产物。

### 安全与外部命令

- `workdir` 只能在约定的项目根范围内使用。
- 外部命令必须通过参数数组调用；禁止拼接 shell 用户输入。
- 未确认的 CLI 事件格式、外部接口或 hook 行为，先探测或阅读源码，再实现。
- 日志中不要记录 token、密钥、凭据、完整敏感 prompt 或其他不必要的敏感内容。
- 本项目不允许向系统 Python 安装任何包。
- 本项目的 Python 运行与测试默认使用 Hermes 自己的虚拟环境：`/home/dontstarve/.hermes/hermes-agent/venv/bin/python`。
- 若需要安装 Python 依赖，只能安装到 Hermes 的 venv；禁止使用全局 `pip`、`pip3`、`python -m pip` 或 `python3 -m pip`。
- 在安装任何 Python 依赖前，必须先向用户说明其必要性、安全性、合理性，以及为什么不能先用标准库或现有依赖解决；获得用户明确同意后才能安装。
- 若某个依赖不适合进入 Hermes venv，应改成独立进程或独立服务集成，而不是直接在插件内 import。

### 开发纪律

- 一个任务只做一个聚焦变更，不顺手扩大 scope。
- 行为变化必须补测试，或在结果中明确说明未补的原因。
- 改动设计边界时，更新 `DESIGN.md` 或 `docs/DECISIONS.md`。
- 不要把展示层判断、会话持久化和 Codex 事件解析耦合在一起。
- 发现更大的改进机会时，先记入 `docs/TASKS.md`，不要混入当前任务直接实现。

## 收尾

结束前：

- 跑与当前任务直接相关的检查。
- 更新 `docs/TASKS.md`，记录：
  - 已完成内容
  - 未完成内容
  - 下一步建议
  - 阻塞点（如果有）
- 如果设计约束、术语或模块边界变化，更新 `docs/DECISIONS.md`。
- 未经用户明确要求，不要提交 commit。

最终回复必须说明：

- 本轮做了什么
- 跑了哪些检查
- 跳过了哪些检查以及原因
- 是否更新了 `DESIGN.md`、`docs/TASKS.md`、`docs/DECISIONS.md`
- 是否产生了新的 `run/` 运行态文件
